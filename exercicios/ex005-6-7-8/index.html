Negrito e Itálico

A tag b de bold pode colocar negrito as coisas porém ela não adiciona nada além do efeito visual. Já a tag strong cria um significado, dando "semântica" a palavra

Um exemplo visual entre bold e strong, visualmente identicos, porém para o código são coisas diferentes

Também existe uma tag de enfase que é dada por "em", deixando esse aspécto destacado
A diferça entre os termos é a mesma que bold e strong para negrito, um é apenas visual, o outro é semântico.
o "i" funciona como o visual e "em" funciona como semântico

A tag de marcador parece um marca-texto de caderno tipo MARK, até que fica bonitin, mas dá pra adaptar melhor usando em CSS uma complexidade interessante.
Como se faz isso?
Simples, usar a tag style dentro da tag mark, isso basicamente "importa" as funções de CSS

É possível colocar um riscado, como se fosse um texto a ser desconsiderado na tag "del", e o oposto vale pra tag "ins", que seria o inserido. Ele é a proporção semântica da tag "u", que seria puramente "sublinado"

Seguindo mesmo ramo porém mais pra operações, seria a tag "sub", que seria de subescrito e "sup", que seria sobrescrito.

Finalizando a manipulação de caracteres, a gente tem a função monoespaçada que é citada por "code", onde uma aplicação mais sentada seria de fato em um código como for(item.index in list->elemento.char){return temp->prox.indexdireito}. Um código inventado mas que esclarece um uso melhor.
O que a tag faz é basicamente com que todos os caracteres assumam o mesmo espaço na tela.
Já se quiser fazer algo que respeite até a identação do seu código é só envelopar a tag code com a tag pre. Fica dessa forma: 


        dynamic valor=23
        if(valor>=14)

        else{
            printf("Ta muito além");
            return false;
        }

Então ele basicamente respeita a distância inserida entre elementos no codigo. Tendo seu 'nome' em pre porque ele é uma pré-formatação
