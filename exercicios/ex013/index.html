INLINE

Não é o ideal

O inline é basicamente fazer o código na própria linha da tag adicionando o style e colocando as modificações

Por que não se usa?

Basicamente ele limita pra caramba as modificações, já que só serve justamente pra tag em especifico que colocou, e não pra outras iguais a ela, então isso faz com que o código possua redundâncias e diversas dificuldades em produzir classes mais fluidas

Diferenças

HTML

Como o HTML busca entregar semântica e sentido as coisas, suas classes e escolhas de tag, mesmo muitas vezes não sendo visualmente diferentes como conversamos entre bold e strong, a forma como ele interpreta é mais relevante aos mecanismos de busca e ordenação de elementos

CSS

O Css por sua vez, não trás significado algum as suas modificações, é puramente visual, cascatas de efeito design, não possuem real diferença ao navegador, apenas a quem vê o estilo, atraindo o visitante.

Não use basicamente

O inline não é util de modo genérico, vai ter certos momentos onde poderá ser um quebra galho, porém ele polue e prejudica a manutenabilidade do código, mesmo que tenha o mesmo efeito visual no usuário

Css Local/Interno

Já a tag Style no head, usada, mas nem tanto

Tudo ali dentro se torna uma espécie de doc interno de css, já é uma grande evolução do inline, porém se apresenta melhor para justamente estilos que são aplicados a apenas um documento html, já que se estivermos falando de um site, cada pagina possui um documento independente, então se for para manipular estilos dessa forma, teria de copiar para cada uma das paginas

A MAIS LINDA!

Link para CSS externo

Dessa forma no head, você coloca uma tag link:css e pode fazer as modificações de estilo em um documento css externo
